django_twitter.models¶

Django Twitter provides a series of abstract models to store data from the Twitter API. If you import these abstract classes and your own app’s models inherit from them, Django Twitter will automatically create a standard set of model fields and relations for you.

Base models¶

class AbstractTwitterBase[source]¶

All django_twitter models inherit from this base class. When a model in your own app, in turn, inherits from a django_twitter model (like AbstractTweet), the code below will intercept the model when it’s initialized, and then look other django_twitter-based models in your own app that should have relationships between each other. django_twitter doesn’t know anything about your app but if you configure your settings.py model correctly, it will be able to connect all your models together at runtime.

static __new__(cls, name, bases, attrs)[source]¶

Overrides Django’s __new__ function and uses it to auto-detect the models in your own app that implement Django Twitter’s abstract models, and auto-creates relations between them at runtime.

class AbstractTwitterObject(*args, **kwargs)[source]¶

A base class for all Twitter models, including a unique Twitter ID and a timestamp reflecting when the object was last updated. Also has a historical boolean field that you can use for your own purposes (e.g. for flagging and skipping outdated accounts during data collection, etc.)

Parameters

  • twitter_id (CharField) – Twitter id. The object’s unique Twitter ID

  • last_update_time (DateTimeField) – Last update time. Last time the object was updated

  • historical (BooleanField) – Historical. Empty flag that you can use to track historical accounts

save(*args, **kwargs)[source]¶

Strings and lowercases Twitter IDs and updates last_update_time every time the object is saved.

get_snapshots(start_date, end_date, *extra_values, **kwargs)[source]¶

Loops over the profile’s snapshots for a given time range, and compiles a Pandas DataFrame of the profile’s dynamic data (e.g. follower counts, description, etc.) Uses linear interpolation to fill in missing days for numeric values, and front-filling for non-numeric values.

Parameters

  • start_date (datetime.datetime or datetime.date) – The start of the date range you want to extract

  • end_date (datetime.datetime or datetime.date) – The end of the date range you want to extract

  • extra_values – If you would like to include additional fields (e.g. foreign keys from related objects) you can pass their names as related objects (e.g. “profile__politician_id”)

  • skip_interpolation – (Optional) Disables the default interpolation. Will keep days with missing values empty.

Returns

Pandas DataFrame of the profile’s snapshots

A template for storing data contained in a tweet.

  • hashtags = models.ManyToManyField(your_app.TwitterHashtagModel, related_name=”tweets”)

  • in_reply_to_status = models.ForeignKey(your_app.TweetModel, related_name=”replies”)

  • quoted_status = models.ForeignKey(your_app.TweetModel, related_name=”quotes”)

Parameters

  • twitter_id (CharField) – Twitter id. The object’s unique Twitter ID

  • last_update_time (DateTimeField) – Last update time. Last time the object was updated

  • historical (BooleanField) – Historical. Empty flag that you can use to track historical accounts

  • created_at (DateTimeField) – Created at. The time/date that the tweet was published

  • links (ArrayField) – Links. Links contained in the tweet

  • media (ArrayField) – Media. Media contained in the tweet

  • text (CharField) – Text. Text extracted from the tweet, including expanded text and text from tweets that it quoted or retweeted (hence why the max length is longer than the twitter size limit

  • profile_mentions_raw (ArrayField) – Profile mentions raw. A list of profile screen names that were mentioned in the tweet

  • language (CharField) – Language. The tweet’s language

  • retweet_count (IntegerField) – Retweet count. Number of times the tweet was retweeted. Note: for tweets that are retweets (but not quote tweets), this count reflects the _original_ tweet’s retweets, not just the retweeted version’s retweets. When someone retweets a retweet that didn’t have any additional commentary, that retweet gets redirected back to the original tweet.

  • favorite_count (IntegerField) – Favorite count. Number of times the tweet was favorited. Note: for tweets that are retweets (but not quote tweets), this count reflects the _original_ tweet’s favorites, not just the retweeted version’s favorites. When someone favorites a retweet that didn’t have any additional commentary, that favorite gets redirected back to the original tweet.

  • json (JSONField) – Json. The raw JSON for the tweet
